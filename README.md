# Grade Clicker — Activity Lifecycle

## Цель работы
Изучить жизненный цикл Activity в Android и научиться правильно обрабатывать переходы между состояниями.

## Функциональность
- Кликер для набора баллов (+5 за клик)
- Изменение изображения при достижении порогов (0, 50, 70, 90, 100)
- Отображение текущей оценки и сообщения
- Счётчик баллов и кликов
- Сохранение состояния при повороте экрана (rememberSaveable)
- Логирование lifecycle-методов

## Контрольные вопросы

**Какие три метода вызываются при запуске приложения?**
onCreate() 
onStart() 
onResume()

**Чем отличается onPause() от onStop()?**
onPause() — Activity теряет фокус, но может быть частично видна. onStop() — Activity полностью не видна.

**Когда вызывается onRestart()?**
При возврате Activity из состояния Stopped (после onStop()).

**Что происходит при повороте экрана?**
Activity уничтожается (onDestroy) и создаётся заново (onCreate).

**В чём разница между remember и rememberSaveable?**
remember сохраняет данные только при рекомпозиции, rememberSaveable — также при configuration changes.

**Почему onCreate() должен вызывать super.onCreate()?**
Для корректной работы системы Android и правильной инициализации Activity.

**Для чего используется класс Log?**
Для вывода отладочных сообщений в Logcat.

**Какой метод вызывается только один раз за время жизни Activity?**
onCreate() — вызывается только при создании Activity.
